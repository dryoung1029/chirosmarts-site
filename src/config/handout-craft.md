# Handout craft — agent brief

> Injected into every Collateral Studio generation prompt. Distilled from
> instructional-design evidence (Bloom's, Mayer's multimedia principles, ADDIE,
> retrieval/spaced practice, ACCME/SACME CE standards) for turning clinical
> training transcripts into purpose-built learning artifacts. The grounding rule
> in the system prompt OVERRIDES anything here: pedagogy and structure come from
> this brief; facts come only from the transcripts.

## The core idea
A handout is NOT a reformatted transcript or slide dump. It is a standalone
learning artifact that must be comprehensible **without the presenter**. Build to
the learning objective and the reader's use-after-session, not to the transcript's
running order.

## Extract these 11 signals from the transcript first
1. **Learning objectives** — "by the end…", "today you'll…" (explicit or implied
   from the opener/closer).
2. **Key terms & definitions** — clinical terminology, defined in plain language.
3. **Core concepts** — the central mechanisms/frameworks (1–3 sentence summaries).
4. **Procedural steps** — numbered/sequential techniques.
5. **Clinical evidence** — studies, guidelines, standards cited.
6. **Case examples** — patient vignettes / illustrative scenarios.
7. **Common errors / practice gaps** — "most practitioners miss…", current-vs-best
   practice contrasts.
8. **Decision criteria** — when to use X vs Y, contraindications, red flags.
9. **Notable quotes** — crystallizing statements worth preserving.
10. **Resources** — books, tools, sites, follow-ups mentioned.
11. **Self-assessment prompts** — questions the instructor posed.

Weight by **emphasis + frequency** ("this is important", "remember this", repeated
points rank higher). Carry terms forward so definitions aren't repeated.

## Cut the noise (Coherence Principle)
Remove housekeeping/logistics, filler ("uh", "does that make sense"), off-topic
tangents, repetitive restatements, social pleasantries, inside jokes.

## Pedagogy to build in
- **Bloom's range.** Don't stop at recall. Review questions must span at least
  three levels: Remembering → Understanding → Applying/Analyzing/Evaluating.
- **Objectives as behaviors (SWBAT / ACCME).** "Practitioner will be able to…"
  describing observable competence, not vague "understand X."
  - ✗ "Understand spinal manipulation indications."
  - ✓ "Identify three contraindications to high-velocity thrust manipulation and
    modify the plan accordingly."
- **Pre-training principle.** Put Key Terms / glossary BEFORE the main content.
- **Concrete over abstract.** Pair every core concept with a brief clinical
  application example. If the transcript lacks one, you may add a clearly
  hypothetical scenario ("Consider a patient who…") — never a false first-person
  anecdote and never invented clinical values, fees, or facts.
- **Signaling & segmenting.** Headers, bold key terms, numbered steps, short
  delineated sections — not long prose blocks.
- **Retrieval practice.** Every artifact includes a self-testing mechanism
  (review questions, prompts).
- **Spaced review.** Close a study guide with a "review again at 24 hours, 1 week,
  1 month" prompt.
- **Practice gaps.** Elevate "what practitioners commonly get wrong" into its own
  Common Errors / Pitfalls section — this is the highest-value CE content.

## Length targets
- Study guide: ~3–6 pages.
- Checklist / job aid: 1–2 pages, every item earns its place.
- Cheat-sheet / quick-reference: 1–2 pages, scannable, point-of-care.

## Quality self-check before finishing
- [ ] Objectives at top, written as behaviors (SWBAT).
- [ ] Every key term defined in plain language.
- [ ] ≥1 clinical application example per core concept.
- [ ] Review questions span ≥3 Bloom's levels (study guide).
- [ ] Filler/noise removed; comprehensible without the presenter.
- [ ] Common errors / practice gaps surfaced.
- [ ] Decision criteria / red flags called out where present.
- [ ] Length appropriate to the format.
- [ ] Nothing fabricated — every fact traces to the transcript.
